# 274. H-Index

Link: https://leetcode.com/problems/h-index/
Tags: Array, Sorting, Counting Sort
Difficulty: Medium

## 題目解釋
給你一個整數陣列 `citations` ，其中 `citations[i]` 表示研究者的第 `i` 篇論文被參照的次數。計算並返回該研究者的 `h` 指數。
h 指數的定義：h 代表“高參照次數”（high citations），一名科研人員的 h 指數是指他（她）的 （`n` 篇論文中）總共有 `h` 篇論文分別被參照了 **至少** `h` 次。且其餘的 `n - h` 篇論文每篇被參照次數 **不超過** `h` 次。

## 限制與條件
- `n == citations.length`
- `1 <= n <= 5000`
- `0 <= citations[i] <= 1000`

## 解法思路
### 思考過程
H指數的最大值不會超過論文總數 `n`。
我們可以利用 **計數排序 (Counting Sort / Bucket Sort)** 的思想：
1. 建立一個大小 `n+1` (或題目限制 1001) 的陣列 `quote`，`quote[i]` 記錄引用次數 **等於** `i` 的論文有多少篇。
   - 如果引用次數超過 `n`（或 1000），可以將其視為 `n`（或 1000），因為這對 H 指數的計算效果一樣（至少貢獻了最大可能的 H）。
2. 從後往前（從高引用次數到低）遍歷 `quote` 陣列，維護變數 `cnt` 記錄「引用次數 >= 當前值 i」的論文總數。
3. `cnt += quote[i]`。
4. 如果 `cnt >= i`，說明我們找到了至少 `i` 篇論文，它們的引用次數都大於等於 `i`，那麼 `i` 就是滿足條件的最大的 H 指數。

### 解法：Counting Sort
> [!IMPORTANT]
> 思路：使用桶排序統計引用分佈，從高到低累計找到最大的 H 值。

### 複雜度分析
| 維度 | 複雜度 | 說明 |
| --- | --- | --- |
| 時間複雜度 | O(n) | 遍歷引用與遍歷桶均為線性時間。 |
| 空間複雜度 | O(n) | 需要一個額外的計數陣列。 |

```cpp
class Solution {
    public:
        int hIndex(vector<int>& citations) {
            int quote[1001]={0};
            int cnt=0;
            for(int i:citations){
                quote[i]++;
            }
            for(int i=1000; i>0; i--){
                cnt+=quote[i];
                if(cnt>=i){
                    return i;
                }
            }
            return 0;
        }
    };
```
